# Design for Player Value

Design from the player task outward. Treat screen space and attention as scarce.

## Require an immediate job

Before adding or retaining an element, state the concrete job it performs for a player:

- choose a map, hero, mode, or strategy
- compare viable options
- understand compatibility, difficulty, or game-version risk
- execute a build accurately
- recover from an empty, invalid, or outdated state
- navigate to the next likely task

Remove the element when its only job is to make the page feel complete, impressive, trustworthy, or product-like.

## Reject common filler

Do not add these by default:

- generic eyebrow copy that names the product category
- slogans that restate the product promise
- paragraphs explaining functionality already obvious from the controls
- inventory totals such as number of guides, maps, heroes, or builds
- vanity statistics presented as proof of quality
- badges such as "verified" or "curated" without actionable criteria or consequences
- extra cards, panels, gradients, or animations used only to occupy space

Metrics earn space only when they change a decision. Prefer contextual counts such as “3 guides for this map” over global totals such as “128 guides available.”

## Follow task-first hierarchy

1. Put the primary player choice or result first.
2. Show decision-critical context beside the relevant option.
3. Defer advanced controls and secondary metadata.
4. Use empty space instead of filler content.
5. Keep copy literal, specific, and short.

For discovery pages, lead with search, filters, or the domain objects themselves. For guide pages, lead with compatibility and execution. Do not insert a marketing hero between the player and the task unless the page genuinely needs onboarding for an unfamiliar workflow.

## Review every change

For each new UI region, ask:

1. What player question does this answer?
2. What decision or action changes because it exists?
3. Is that value already obvious elsewhere?
4. Would removing it make the task harder?

If questions 1 or 2 lack a specific answer, remove it. If question 4 is no, strongly prefer removal.

When handing off work, call out any intentionally omitted content where this skill prevented non-functional UI from being added.
